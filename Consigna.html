Condiciones generales

El proyecto integrador es grupal y se realiza de a grupos de 2 o 3 personas. Los alumnos pueden elegir sus compañeros de trabajo libremente. Deben confirmar los grupos de trabajo vía Slack hacia la cátedra para el 15/3 En caso de tener algún conflicto es fundamental que lo avisen previamente.

 La fecha  de la primera entrega del proyecto integrador será el  lunes 26 de abril.  Las presentaciones se harán en forma grupal vía zoom. Queda a criterio de los alumnos si desean agregar recursos o soporte (como PPTs) para sus exposiciones.

Se espera durante la presentación una demo de la página web y un breve recorrido por el código destacando aspectos que fueron dificultosos durante el desarrollo. Durante la presentación también se harán preguntas a los alumnos que esperamos se respondan de manera individual. La nota de la presentación es individual y mientras que la del trabajo en si es grupal.

La entrega se realizará pasando vía slack el link al repositorio del proyecto en GIthub.

Es importante que utilicen GIT durante el proyecto y no únicamente para la entrega. Estaremos observando el commit log y quienes realizan dichos commits. Esto afecta tanto a la nota del trabajo como la nota de concepto.

Para el desarrollo del proyecto se le dará a cada grupo un proyecto base que puedes descargar aquí. Contará con la estructura HTML y el estilo del sitio en CSS. Cada grupo deberá utilizar esta base y completarla con las consignas del proyecto.

Consigna General

El proyecto consiste en armar un catálogo de productos en donde los usuarios podrán ver y comentar cada producto. A diferencia de un ecommerce, un catálogo de productos no cuenta con los datos de precios ni con la funcionalidad de carrito de compras.

Como primera instancia deberán migrar el proyecto del formato entregado visto en Programación I a un proyecto planteado con Node y Express.

La migración debe considerar el patrón MVC, es decir, deberán crear las rutas necesarias, con sus controladores y vistas por recurso.

La estética del sitio debe mantener una línea en común a través de todas sus páginas. Header y footer deben ser comunes en todas las páginas.

Deben revisar que todas las páginas estén vinculadas entre sí correctamente así cómo el envío de formularios.

El footer deberá ser actualizado con la información correspondiente de cada grupo.

Organización de los equipos

Cada equipo deberá administrar la distribución, asignación y cumplimiento de tareas mediante un tablero Trello.

El equipo deberá designar un responsable del seguimiento del tablero para que se mantenga  actualizado. Quien sea designado responsable velará porque cada miembro del equipo cargue y mantenga actualizadas sus tareas. No será él (el responsable designado) quien deba cargar o actualizar las tareas. En caso de incumplimientos deberá reclamar o exigir a cada miembro del equipo que cumpla con cargar las tarjetas y mantenerlas actualizadas.

Consignas

Punto 1 - Migración a Node y Express

Deberán migrar todo el proyecto utilizando Node y Express.

Tener en cuenta los siguientes puntos:

  1. Crear las rutas necesarias para que todas las páginas del proyecto funcionen. Las rutas deberán utilizar controladores.
  2. Crear los controladores que se encarguen de la lógica de cada página. Los controladores deberán renderizar vistas.
  3. Crear las vistas necesarias utilizando EJS.
  4. Tener en cuenta que los hipervínculos ya no redirigirán a un archivo .html, sino a una ruta como las mencionadas en primer punto. Lo mismo aplica a los formularios.
  5. Modificar las rutas de los links de CSS e imágenes de cada EJS para que funcione correctamente con la estructura que plantea Express (el uso de la carpeta /public).
  6. Se utilizará cómo criterio de evaluación la prolijidad en la organización de los archivos de acuerdo a los recursos, es decir, valoraremos en qué archivo se encuentra cada ruta y método controlador dependiendo del recurso al que responda.
  7. Completar las siguientes páginas según los datos que vaya a tener el usuario en el sitio. Es importante que estos datos coincidan con los que aparecen en la base de datos (Punto 3):
  1. Registración → Completar la vista del formulario
  2. Login → Completar la vista del formulario
  3. Mi Perfil → Completar la sección superior según los datos que le pidan al usuario.
  4. Editar mi perfil → Completar la vista del formulario.
  5. Agregar producto → Completar la vista del formulario

  1. El header y el footer deben estar armados de modo tal que al modificarlo se actualicen en todas las páginas.
  2. Para esta entrega las páginas que precisen datos van a contener datos ficticios ya que no contamos con el acceso a una base de datos. Por ejemplo, en la página de “Mi Perfil” habrá datos de 1 usuarios escritos a mano en la vista.
  3. Las páginas “Mi perfil”, “agregar producto” y “editar mi perfil” son páginas a las que el usuario únicamente accede estando logueado por lo que deberán aplicar las variantes visuales que indiquen al usuario que efectivamente está logueado. (Archivo headerLogueado.html).

Punto 2 - Personalización del sitio

Deberán modificar el proyecto base proporcionado cumpliendo los siguientes requisitos:

  1. Cada grupo elegirá un tema o tipo de productos para su catálogo y deberá adaptar la estética y contenidos. Por ejemplo: catálogo de autos, de relojes, de celulares, de productos de computación, de libros, de mascotas, de muebles, de cuadros, etc.
    Cada equipo deberá reemplazar los productos base por al menos la misma cantidad de la temática que elijan.
  2. Tipografía:
  1. Elegir al menos 3 tipografías de texto para aplicar en el sitio.
  2. Cada título principal deberá tener una tipografía que destaque de los demás textos del sitio.
    Deberá mantenerse la misma tipografía en las diferentes vistas.
  3. Utilizar una tipografía que destaque los hipervínculos de los demás textos del sitio.
    Deberá mantenerse la misma tipografía en las diferentes vistas.

  1. Nombre del sitio:
  1. Deberán definir un nombre que represente al sitio e implementarlo en el header.
  2. El nombre debe persistir en todas las páginas del sitio.
  3. El nombre deberá tener una tipografía única.
  1. Paleta de colores:
  1. Deberán cambiar la paleta de colores utilizada en el sitio para darle un estilo distintivo y adecuado a la temática elegida..

Punto 3 - Base de datos

Deberán crear una base de datos que con:

  1. Una tabla de usuarios donde cada equipo decidirá qué datos tendrá su usuario. Cómo mínimo debe contar con:
  1. Un campo de texto.
  2. Un campo de fecha.
  3. Un campo numérico (además del id de la clave principal).

  1. Una tabla de productos que contenga al menos :
  1. ID del usuario que lo cargó.
  2. URL de la imagen.
  3. Nombre del producto
  4. Fecha de creación.
  5. Las claves foráneas que correspondan.

  1. Una tabla de comentarios que tenga al menos:
  1. ID del producto sobre el cual se está comentando.
  2. ID del usuario que creó el comentario.
  3. Texto del comentario.
  4. Fecha de creación.
  5. Las claves foráneas que correspondan

Deberán insertar al menos 5 usuarios, 10 productos y 4 comentarios para cada producto.

Los campos de las tablas deben coincidir con la información solicitada en cada formulario creado en el punto 1.7.

Deberán entregar un archivo con las sentencias SQL para la creación de las tablas y la inserción de datos o podrán optar por entregar un archivo con el dump de la base de datos para que los profesores podamos importarla y evaluar.
